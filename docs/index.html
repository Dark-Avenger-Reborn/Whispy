Built for modern Python workflows

pip install whispy-client

# Use it anywhere
from whispy_client import remote

requests = remote("requests")

remote()

Import a package from the Whispy CDN at runtime and return the module object.

remote(
  package: str,
  *,
  module:  str  = None,
  version: str  = None,
  deps:    bool = False,
  host:    str  = None,
) -> ModuleType

Parameters

NameTypeDescription
packagerequired str PyPI distribution name. "requests".
moduleoptional str Import name when it differs from the distribution name, e.g. module="bs4" for "beautifulsoup4".
versionoptional str Explicit version string. Overrides any spec embedded in package.
depsoptional bool If True, resolve and bundle all transitive dependencies. Defaults to the global configure() setting.
hostoptional str CDN base URL override for this call only. Useful for testing against a local server.

Common name mismatches

bs4      = remote("beautifulsoup4", module="bs4")
PIL      = remote("pillow",         module="PIL")
yaml     = remote("pyyaml",         module="yaml")
dateutil = remote("python-dateutil", module="dateutil")
cv2      = remote("opencv-python",  module="cv2")

configure()

Set global defaults for all subsequent remote() calls.

configure(
  *,
  host:    str  = None,   # CDN base URL
  deps:    bool = None,   # auto-fetch dependencies
  verbose: bool = None,   # print progress
)

You can also configure via the WHISPY_HOST environment variable:

WHISPY_HOST=http://localhost:5000 python my_script.py

GET /get_package

Resolve, verify, and return a zip containing the package (and optionally its deps) ready for sys.path insertion.

GET /get_package
ParamRequiredDescription
namerequiredPyPI distribution name
tagsrequiredComma-separated PEP 425 tags, most-specific first
versionoptionalExact version. Omit to resolve latest.
depsoptional1 to include transitive dependencies

Response headers include X-Whispy-Version-Resolved (actual version served) and X-Whispy-Manifest (JSON array of all packages in the zip).

Self-hosting

Run your own Whispy instance for internal tools, air-gapped environments, or maximum control. The server is a simple Flask app with zero external dependencies.

# Install server dependencies
cd server
pip install -r requirements.txt

# Run locally (development)
python app.py --debug

# Or with gunicorn (production)
gunicorn app:app -w 4 -b 0.0.0.0:8000

Then point your client at your instance:

configure(host="http://your-server:8000")

See the server/app.py source for configuration options like WHISPY_CACHE_DIR, WHISPY_MAX_CACHE_MB, and REDIS_URL.

Use Cases

Built for these moments

🎓 Education

Students run Python scripts without pip, venvs, or permissions. Share a single .py file and it just works on any machine.

from whispy_client import remote
matplotlib = remote("matplotlib", deps=True)
# No "install matplotlib first"

☁️ Serverless / Lambda

Hydrate dependencies at cold-start instead of bundling them into your deployment package. Smaller deploys, faster CI.

# In your Lambda handler
    configure(host="https://whispycdn.dev")
    httpx = remote("httpx", deps=True)

🔌 Plugin systems

Load capabilities at runtime based on user config. No need to bundle every possible plugin — fetch on demand.

# Load the plugin the user actually needs
plugin_name = config["export_format"]
plugin = remote(f"whispy-plugin-{plugin_name}")

🧪 Ephemeral scripts

One-off data scripts that need a heavy library once. Import it, use it, and leave the system exactly as you found it.

pandas = remote("pandas", deps=True)
df = pandas.read_csv("data.csv")
# pandas vanishes when process exits
